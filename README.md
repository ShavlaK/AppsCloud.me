# Price Updater для Telegram

Автоматическое обновление прайс-листа в Telegram канале каждый час.

## Установка

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Настройте `config.json`:
   - `source_bot_token` - токен бота-источника (откуда парсим цены)
   - `your_bot_token` - токен вашего бота (который публикует в канал)
   - `channel_id` - ID или username вашего канала (например, `@my_channel`)
   - `price_markup` - сумма наценки на каждую позицию
   - `price_message_id` - ID сообщения с прайсом (заполнится автоматически)

## Запуск

```bash
python main.py
```

После запуска веб-интерфейс будет доступен по адресу: **http://localhost:8000**

## Веб-интерфейс (Dashboard)

Панель управления предоставляет:

### 📡 Статус
- Статус планировщика (работает/остановлен)
- Статистика базы данных (количество снимков, позиций, размер)
- Расписание задач
- Кнопки: старт/стоп планировщик, ручное обновление

### ⚙️ Конфигурация
- Редактирование токенов ботов
- Настройка наценки
- Изменение расписания
- Настройка User API (Telethon)

### 📝 Логи
- Просмотр логов в реальном времени
- Фильтрация по файлам (основной, ошибки, планировщик, Telegram)
- Настройка количества строк

### 📈 Отчёты
- Генерация ежемесячных отчётов
- Список сохранённых отчётов
- Просмотр детальной статистики

## REST API

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/api/status` | Статус системы |
| GET | `/api/config` | Текущая конфигурация |
| POST | `/api/config` | Обновить конфигурацию |
| POST | `/api/update` | Ручное обновление прайса |
| POST | `/api/scheduler/start` | Запустить планировщик |
| POST | `/api/scheduler/stop` | Остановить планировщик |
| POST | `/api/report` | Сгенерировать отчёт |
| GET | `/api/reports` | Список отчётов |
| GET | `/api/logs` | Просмотр логов |

## Как это работает

- Скрипт запускается и ждёт 11:30
- Каждый час до 19:30 парсит цены из бота-источника
- Добавляет наценку
- Обновляет сообщение в канале
- Сохраняет снимок в SQLite (хранится 30 дней)

## Настройка расписания

В `config.json` можно изменить:
- `start_hour` / `start_minute` - время начала
- `end_hour` / `end_minute` - время окончания
- `interval_hours` - интервал обновлений

## Структура логов

```
logs/
├── price_updater.log   # Все логи
├── errors.log          # Только ошибки
├── scheduler.log       # События планировщика
└── telegram.log        # Логи Telegram API
```

## Хранение данных

- `prices.db` — SQLite база с историей цен (30 дней)
- `prices.json` — последний прайс (для совместимости)
- `reports/` — JSON файлы ежемесячных отчётов
